# オブジェクト指向プログラミング学習

このディレクトリは、YouTubeの動画を参考にしながら、オブジェクト指向プログラミング（OOP: Object-Oriented Programming）について理解を深めるための学習用ディレクトリです。

## 📚 学習の目的

オブジェクト指向プログラミングは、コードを整理し、再利用しやすく、保守しやすくするための重要なプログラミングの考え方です。このディレクトリでは、実際のコード例を通じて、オブジェクト指向の概念を学びます。

### オブジェクト指向とは？

オブジェクト指向は、プログラムを「物（オブジェクト）」として捉える考え方です。例えば、ポケモンのゲームでは、各ポケモンを「オブジェクト」として扱い、それぞれが「名前」「HP」「攻撃力」などの属性（プロパティ）と「攻撃する」「ダメージを受ける」などの動作（メソッド）を持ちます。

## 📁 ファイル構成

### `no_oop_pokemon.py`

オブジェクト指向を使わずに実装したポケモンバトルゲームのコードです。

**特徴：**

- 辞書型（`dict`）を使ってポケモンのデータを表現
- 関数を使って処理を実装
- データと処理が分離されている

**このコードから学べること：**

- オブジェクト指向を使わない場合のコードの書き方
- データと処理が分離されていることによる問題点

### `pokemon.py`

オブジェクト指向を使って実装したポケモンのクラス定義です。

**特徴：**

- `Pokemon`クラス：すべてのポケモンの基本となるクラス（基底クラス）
- `Pikachu`クラスと`Hitokage`クラス：`Pokemon`クラスを継承した具体的なポケモンクラス
- プロパティ（`@property`）を使ってデータへのアクセスを制御
- メソッドを使ってポケモンの動作を定義

**このコードから学べること：**

- クラスの定義方法
- 継承（Inheritance）の概念
- カプセル化（Encapsulation）の実装
- ポリモーフィズム（Polymorphism）の実装（`attack_message`メソッド）

### `game.py`

バトルゲームのロジックを管理するクラスです。

**特徴：**

- `Game`クラス：2匹のポケモンを受け取り、バトルを進行する
- 素早さ（speed）に基づいて攻撃順を決定
- 勝敗判定と結果表示

**このコードから学べること：**

- クラスを使ったゲームロジックの実装
- オブジェクト間の相互作用

### `oop_pokemon.py`

オブジェクト指向版のメイン実行ファイルです。

**実行方法：**

```bash
python oop_pokemon.py
```

このファイルを実行すると、オブジェクト指向版のポケモンバトルが開始されます。

## 🎯 学習の進め方

1. **まず、両方のコードを実行してみる**
   - `no_oop_pokemon.py`を実行して、オブジェクト指向を使わない実装を確認
   - `oop_pokemon.py`を実行して、オブジェクト指向版の実装を確認

2. **コードを読んで理解する**
   - `no_oop_pokemon.py`：各関数がどのような役割を持っているか、データがどのように管理されているか
   - `pokemon.py`：クラスの定義、継承、プロパティの使い方
   - `game.py`：クラスを使ったゲームロジックの実装

3. **両方のコードを比較する**
   - 同じ機能を実現するための2つのアプローチを比較
   - オブジェクト指向のメリット（コードの再利用性、保守性、拡張性）を理解する

## 💡 学習のポイント

### オブジェクト指向の主な概念

1. **クラス（Class）**
   - オブジェクトの設計図のようなもの
   - 例：ポケモンというクラスを定義することで、様々なポケモンを作成できる

2. **インスタンス（Instance）**
   - クラスから作られた実際のオブジェクト
   - 例：ピカチュウというインスタンス、ヒトカゲというインスタンス

3. **属性（Attribute/Property）**
   - オブジェクトが持つデータ
   - 例：名前、HP、攻撃力など

4. **メソッド（Method）**
   - オブジェクトが実行できる動作
   - 例：攻撃する、ダメージを受けるなど

### このコードで学べるオブジェクト指向の概念

1. **継承（Inheritance）**
   - `Pikachu`クラスと`Hitokage`クラスは`Pokemon`クラスを継承しています
   - 共通の機能（HP、攻撃力など）を基底クラスに定義し、個別の特徴（攻撃メッセージなど）を子クラスで定義
   - コードの重複を減らし、保守性を向上させます

2. **カプセル化（Encapsulation）**
   - プライベート属性（`_name`、`_hp`など）を使って、データへの直接アクセスを制限
   - プロパティ（`@property`）を使って、データの取得と設定を制御
   - データの整合性を保つことができます（例：HPが負の値にならないようにする）

3. **ポリモーフィズム（Polymorphism）**
   - `attack_message`メソッドが、`Pikachu`と`Hitokage`で異なる動作をします
   - 同じメソッド名でも、クラスによって異なる実装を持つことができます
   - コードの柔軟性と拡張性を高めます
